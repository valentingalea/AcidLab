17 · Generative acid

Stop drawing patterns — grow them. Two small algorithms (Euclidean rhythms + a scale random-walk) write endless acid loops. This is how you score a game.

drums · euclidean — each slider spreads N hits evenly over 16 steps
acid line · random walk — generated notes; 🎲 rolls a new one

Slide wander to 1 for a tight, root-hugging line or 4 for a leaping one; density for how busy; the rest are probabilities. Hit 🎲 until you like it.

idea 1Rules + dice = a composer

A hand-drawn loop is one loop. A generator is infinite loops — you write the rules, then let controlled randomness fill in the notes. The catch is taste: pure random sounds like noise; the art is constraints that keep it musical. Two tiny algorithms do almost all the work in dance music, and they're both a few lines. This whole page is those two functions, wired to the instrument you built.

idea 2Euclidean rhythms — spread the hits evenly

Ask a simple question: "place K hits into N steps, as evenly spaced as possible." The answer — a Euclidean rhythm — turns out to be a huge chunk of the world's drum patterns. E(4,16) is four-on-the-floor. E(3,8) is the Cuban tresillo. E(5,8), E(7,16)… are Afro-Cuban, Balkan, techno grooves. One knob per drum (how many hits) and you get real rhythms, no drawing.

Drag the kick / hat sliders and watch the hits redistribute to stay even. That's the entire drum machine here: five Euclidean patterns layered.

// CS corner: the exact-even placement is Bjorklund's algorithm, but a one-liner gets the same spacing: hit[i] = (i·K) mod N < K — a Bresenham line, a hit whenever the running total rolls over. K=0 → silence, K=N → every step, and it always lands a hit on step 0 (the downbeat).

idea 3Melody — a random walk on a scale

For the bassline, pure random notes sound drunk. The fix is a random walk: start on the root and each step move a small distance up or down the scale — not to a random note, but a neighbour or two. Because every note is snapped to the scale mask (from prologue P2 / lesson 13) it can't sound wrong, and because the steps are small it has a melodic contour instead of chaos.

The knobs are the walk's parameters: wander = max step size (1 = creeps, 4 = leaps), density = note-vs-rest, and octaves / accents / slides are just probabilities sprinkled on top. Change the scale to Phryg dom and the same generator writes eastern/"Arabic" acid.

// CS corner: 
let deg = 0;                              // scale degree, start on the root
for (let i=0; i<16; i++) {
  if (rand() < restProb) { note[i] = REST; continue; }
  deg = clamp(deg + randStep(±wander), 0, scale.length-1);   // the walk
  note[i]   = scale[deg];                       // snap to the mask → always in key
  accent[i] = rand() < accProb;                 // probability layers
  slide[i]  = rand() < slideProb;
}
That's the composer. Constrained randomness — a bounded walk over a mask — is the core of nearly all generative music.

idea 4Scoring the game

This is the Module-6 goal made real: the arcade game needs endless, non-repetitive acid without shipping audio files. So it ships these two functions and a seed. Same seed → same loop (reproducible); new seed → a fresh loop, forever, for kilobytes of code. Feed the seed from the level, the score, the player's position — and the soundtrack reacts. You now have a synth and a composer; lesson 18 is about rendering these loops efficiently and getting them into the game.
