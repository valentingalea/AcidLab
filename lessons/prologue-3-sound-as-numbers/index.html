Section 1A sound is just an array

P1 said sound is air wiggling. A microphone turns that wiggle into a voltage, and the computer measures that voltage tens of thousands of times per second, storing each measurement as a number between −1 and +1. That's it. That's digital audio:

sound = float[]   (one measurement is called a sample; playback = feed the numbers to the speaker at the same speed)

Every MP3, every synth plugin, every lesson in this course boils down to producing that array. Below: a real wiggle (green) and the samples the computer actually keeps (dots). Drag to sample it more or less often:

samples per wiggle 12.0

Plenty of dots per wiggle — the orange reconstruction matches the real wave.

Section 2The speed limit — sample rate & aliasing

How often you measure is the sample rate. CDs measure 44,100 times a second (44.1 kHz); pro audio often 48 kHz. Why those numbers? Slide the slider above below 2 samples per wiggle and watch: the dots still sit on the wave, but they also fit a much slower wave — and the computer, seeing only dots, picks the slow impostor. That failure is called aliasing.

So the rule (the Nyquist limit): you need more than 2 samples per wiggle — a sample rate of 44,100 can only represent wiggles up to 22,050 Hz. Human ears give out around 20,000 Hz, which is exactly why CD chose 44,100: the whole audible range, plus margin. Hear aliasing happen:

In the last demo the tone climbs toward 7,500 Hz but the sample rate is 8,000 — past 4,000 Hz (the Nyquist limit) the pitch bounces off the ceiling and comes back down. Frequencies over the limit don't disappear; they reflect. This will matter again when we generate waveforms.

// CS corner: a tone at frequency f sampled at rate fs comes out at f − round(f/fs)·fs — the spectrum wraps around, like integer overflow for pitch.

Section 3Bit depth — how precisely we write numbers down

Sample rate is how often we measure; bit depth is how precisely we store each measurement. CD quality is 16-bit: each sample is one of 65,536 levels. Store fewer bits and the wave gets rounded to a staircase — you hear the rounding error as grit and hiss:

bits per sample 3 bits = 8 levels

Same melody, decreasing precision:

That crunch is a beloved effect — the "bitcrusher". Old samplers and game consoles (8-bit, hence "chiptune") had it for free. We may abuse it on drums later.

Section 4Buffers, blocks & latency

One last mechanical fact. The sound card doesn't ask for samples one at a time — it asks for a block of them ("fill this buffer of 128 samples, you have 2.7 ms"). Audio code lives under a hard real-time deadline: miss one block and everyone hears the glitch. This is why audio runs on its own high-priority thread, and why Web Audio's processing graph feels different from normal JS.

Latency = how many blocks are queued between your code and the speaker. A few queued blocks × 2.7 ms is why software instruments feel "instant" below ~10 ms and "mushy" above ~30 ms.

Section 5 ★Make sound from a for-loop

The payoff. You know sound is an array of −1..+1 numbers, so: allocate one second of samples, fill it with a sine formula, hand it to the speaker. This is the entire program — and the ▶ buttons below run exactly this loop with different formulas:

// one second of A-440, from scratch
const rate = ctx.sampleRate;                  // e.g. 48000 on this machine
const buf  = ctx.createBuffer(1, rate, rate); // 1 channel, 1 second
const s    = buf.getChannelData(0);           // a plain Float32Array
for (let n = 0; n < s.length; n++)
  s[n] = Math.sin(2 * Math.PI * 440 * n / rate) * 0.2;
const src = ctx.createBufferSource();
src.buffer = buf; src.connect(ctx.destination); src.start();

Hear the fizzy edge on the square and saw, especially up high? Their sharp corners contain harmonics above the Nyquist limit from section 2, which fold back down as inharmonic hash — aliasing again. Real synth oscillators (like OscillatorNode in lesson 00) are band-limited: they build the wave only from harmonics below the limit. Build the naive one first, then appreciate the fix.

// CS corner: what you just did is a lookup-table-free software oscillator. The whole course is variations on this loop: filters transform s[n] using its neighbours, envelopes multiply it by a slowly-changing curve, a sequencer decides which f to use when. In lesson 04 we'll write this loop live, 128 samples at a time, inside an AudioWorklet.

Section 6Recap — the whole page in five lines

· Digital sound = a Float32Array of −1..+1 samples, played back at a fixed rate.
· Sample rate caps the highest wiggle at rate/2 (Nyquist); go over and pitch folds back (aliasing).
· Bit depth = precision per sample; fewer bits = staircase = crunch (the bitcrusher / chiptune sound).
· Audio is computed in blocks (128 samples) under a real-time deadline; queued blocks = latency.
· Therefore: a for-loop with a formula is a synthesizer. Everything else is better formulas.
